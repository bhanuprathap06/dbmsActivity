Set Operations.

Master UNION, UNION ALL, INTERSECT and MINUS — the four set operations that combine query results in SQL.

Set Operation 01

UNION

Combines the result sets of two or more SELECT statements and removes all duplicate rows from the output. Both queries must return the same number of columns with compatible data types.

Table A — Employees_India

SELECT EMP_ID, NAME, DEPT FROM Employees_India UNION SELECT EMP_ID, NAME, DEPT FROM Employees_US; -- Priya (102) appears in both tables but shows only once

Removes DuplicatesIdentical rows from both queries are merged into one.
Column Count Must MatchBoth SELECTs need the same number of columns.
Compatible TypesCorresponding columns must have compatible data types.
⚠ Slower than UNION ALLDeduplication adds an extra sort/hash step internally.
Set Operation 02

UNION ALL

Combines result sets of two SELECT statements and retains every row including duplicates. Because no deduplication is performed, it's significantly faster than UNION for large datasets.

Table A — Sales_Q1
SALE_IDPRODUCTAMOUNT
S01Laptop₹80,000
S02Phone₹25,000
Table B — Sales_Q2
SALE_IDPRODUCTAMOUNT
S02Phone₹25,000
S03Tablet₹40,000
union_all_query.sql
SELECT SALE_ID, PRODUCT, AMOUNT FROM Sales_Q1 UNION ALL SELECT SALE_ID, PRODUCT, AMOUNT FROM Sales_Q2; -- Phone (S02) from Q1 and Q2 both appear in results
Result — 4 rows total  ·  Phone / S02 appears twice

Keeps All RowsEvery row from both queries is included — even identical ones.
Fastest Set OpNo dedup step makes it the most performant option.
Use for AggregationIdeal when summing/counting across multiple period tables.
⚠ Intentional DuplicatesMake sure duplicates in output are expected and desired.
Set Operation 03

INTERSECT

Returns only the rows that are common to both result sets — the overlap. Useful for finding shared records between two datasets. Duplicates are automatically removed.

Table A — Premium_Customers
CUST_IDNAME
C01Ananya
C02Karan
C03Meena
Table B — Newsletter_Subscribers
CUST_IDNAME
C02Karan
C03Meena
C04Dev
intersect_query.sql
SELECT CUST_ID, NAME FROM Premium_Customers INTERSECT SELECT CUST_ID, NAME FROM Newsletter_Subscribers; -- Customers who are BOTH premium AND subscribed to newsletter
Result — 2 rows  ·  Only customers present in both tables
CUST_IDNAME
C02Karan
C03Meena
Common Rows OnlyOnly rows existing in both SELECT results are returned.
Removes DuplicatesLike UNION, duplicate rows are automatically eliminated.
Great for MatchingFind users, customers, or records present in two datasets.
⚠ No MySQL supportMySQL lacks native INTERSECT — simulate with INNER JOIN + DISTINCT or an IN subquery.
Set Operation 04

MINUS / EXCEPT

Returns rows from the first query that do not appear in the second query result. Oracle calls it MINUS; PostgreSQL and SQL Server use EXCEPT.

Table A — All_Students
STU_IDNAME
201Aarav
202Bhavna
203Chirag
204Diya
Table B — Exam_Appeared
STU_IDNAME
201Aarav
203Chirag
minus_except_query.sql
-- Oracle SELECT STU_ID, NAME FROM All_Students MINUS SELECT STU_ID, NAME FROM Exam_Appeared; -- PostgreSQL / SQL Server SELECT STU_ID, NAME FROM All_Students EXCEPT SELECT STU_ID, NAME FROM Exam_Appeared;
Result — 2 rows  ·  Students who did NOT appear in the exam
STU_IDNAME
202Bhavna
204Diya
Left Difference OnlyReturns rows unique to the FIRST query only.
Order MattersA MINUS B ≠ B MINUS A — results depend on query order.
Removes DuplicatesOutput is distinct even if the first table has duplicate rows.
⚠ MINUS vs EXCEPTOracle → MINUS  |  PostgreSQL / SQL Server → EXCEPT
